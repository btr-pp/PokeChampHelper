# Pokex 更新日誌

## 最新更新 v2.0

### 移除功能

- ❌ **PWA 支援** — 移除 manifest.json、sw.js、Service Worker 相關設定
  - 現在為純靜態網頁應用
  - 不支援離線模式和桌面安裝
  - 簡化部署，可直接用 `file://` 打開

### 新增功能

#### 1. 隊伍分享（導出/匯入）✨

- **複製隊伍** — 點擊「📋 複製隊伍」，以 JSON 格式複製到剪貼簿
- **匯入隊伍** — 點擊「📥 匯入隊伍」，貼上朋友的隊伍 JSON
- 支援跨裝置、跨設備分享隊伍配置

**JSON 格式示例：**

```json
[
  {
    "name": "噴火龍",
    "types": ["fire", "flying"],
    "moves": ["fire", "flying", "dragon"]
  },
  { "name": "皮卡丘", "types": ["electric"], "moves": ["electric", "steel"] }
]
```

#### 2. 攻擊招式屬性支援 🎯

- 新增成員時，**選完屬性後立即可選擇招式**（最多 4 個）
- 招式選擇區為可選填，可不選
- 隊伍分析結果會顯示每隻成員的招式涵蓋
- 導出隊伍時包含招式資訊，分享時保留招式配置

**使用流程：**
1. 在「我的隊伍」選擇成員屬性（1~2 個）
2. 屬性選完後，下方自動出現「選擇攻擊招式屬性」區塊
3. 點選已學招式（可選，最多 4 個）
4. 按「加入隊伍」確認

**JSON 格式示例：**
```json
{"name":"噴火龍","types":["fire","flying"],"moves":["fire","flying","earthquake"]}
```

#### 3. 屬性查詢頁面改進 ⚡

- ✅ **即時顯示結果** — 點擊任何屬性後立刻計算並顯示查詢結果
- 無需按查詢按鈕，選擇即顯示
- 點擊屬性標籤可快速移除，再次點擊其他屬性繼續查詢
- 「清除」按鈕只在有選擇時出現，點擊清空所有選擇和結果

## 使用說明

### 屬性查詢（即時模式）
1. 在「屬性查詢」分頁點擊屬性按鈕
2. **立即在下方顯示弱點、抵抗、免疫結果**
3. 最多選 2 個屬性（組合查詢）
4. 點「清除」按鈕清空選擇

### 新增隊伍成員並設定招式
1. 在「我的隊伍」輸入寶可夢名稱（選填）
2. 選擇該成員的屬性（1~2 個）
3. **屬性選完後，自動出現招式選擇區**
4. 選擇已學的攻擊招式（最多 4 個，可不選）
5. 點「加入隊伍」確認

### 隊伍分享步驟
1. 在「我的隊伍」確認隊伍配置正確（含招式）
2. 點擊「📋 複製隊伍」，自動複製 JSON 到剪貼簿
3. 分享給朋友（可複製到 Line / Discord / 郵件等）
4. 朋友在「我的隊伍」選擇要導入的隊伍編號
5. 點擊「📥 匯入隊伍」，貼上接收到的 JSON
6. 確認導入 ✅（招式資訊會一起導入）

### 隊伍分析（含招式覆蓋）
1. 建立或編輯隊伍成員（含招式）
2. 點擊「分析隊伍弱點與攻略」
3. 查看：
   - 各成員的弱點屬性及倍率
   - 隊伍共同弱點（危險屬性）
   - 推薦攻擊招式屬性
4. 下方出現「即時對手分析」，選擇對手屬性即看排名

## 技術改動

### 隊伍數據結構

```javascript
// 舊版本
teams[0][0] = { name: "噴火龍", types: ["fire", "flying"] };

// 新版本
teams[0][0] = { name: "噴火龍", types: ["fire", "flying"], moves: [] };
```

### 向後相容性

- 自動遷移舊版本的隊伍數據
- 舊的隊伍資訊不會遺失，會自動轉換成新格式

---

## 已知限制

- 屬性查詢目前只支援查詢防禦倍率，不顯示攻擊倍率
- 隊伍分析成員排序基於對手屬性，不顯示速度等素質
- 招式編輯只能在新增時設定，或透過導入/localStorage 編輯
- 每隊最多 4 隻成員，最多 3 個隊伍（考慮到簡潔性）
