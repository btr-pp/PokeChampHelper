# Pokex 使用者說明書

> 寶可夢屬性相剋查詢與對戰隊伍分析工具

Pokex 是一個純前端單頁工具，用於快速查詢寶可夢屬性弱點、建立對戰隊伍、並針對實際對手提出上場建議。資料會儲存在瀏覽器的 localStorage，不需要登入或網路連線即可使用。

> 💡 使用 App 時，按右下角的 📖 圖示即可隨時開啟本說明書。

## 2. 屬性查詢

### 兩種輸入方式

**（A）直接搜尋寶可夢**

1. 在搜尋欄輸入中文、英文或全國編號（如 `皮卡丘`、`Pikachu`、`25`）
2. 下拉清單會顯示最多 8 個建議
3. 使用 `↑` `↓` 鍵選擇、`Enter` 確認，或直接點選
4. 所選寶可夢的屬性會自動帶入，立即顯示結果

**（B）手動點選屬性**

直接在「或手動選擇屬性」區塊點屬性按鈕，最多選 2 個，選完立即計算。

### 結果區塊

每張卡片會列出倍率分組：

- 🔥 **弱點** — ×2（普通弱點）與 ×4（雙重弱點）
- 🛡️ **抵抗** — ×½ 與 ×¼
- 🚫 **免疫** — ×0（完全不受傷害）

### 操作細節

- 選屬性即時運算，不需按「查詢」按鈕
- 按屬性標籤上的 `×` 可移除該屬性
- 「清除」按鈕只有在已選屬性時才會出現

## 3. 我的隊伍

### 隊伍結構

- 總共 **3 組隊伍**（上方分頁切換），每隊獨立儲存
- 每隊最多 **6 隻**寶可夢
- 當隊伍超過 3 隻時，會自動出現「對戰選角」模式（見 [§4.1](#41-對戰選角6-選-3)）

### 3.1 新增成員

在「新增成員」區塊依序：

1. **（選填）搜尋寶可夢名稱**
   - 從資料庫中選取會自動帶入：屬性、可學招式屬性（最多 4 個）、種族值、Mega 進化表
   - 可學的招式屬性之外的選項會被鎖住（灰色）
2. **選擇屬性**（最多 2 個）
   - 若已透過搜尋帶入，可手動修改
3. **選擇攜帶招式屬性**（選填，最多 4 個）
   - 選完屬性後此區才會出現
   - 若是從資料庫選取的寶可夢，只能選它「學得到」的招式屬性
4. 按 **加入隊伍**

> 無名稱的成員會以屬性組合顯示，例如「火/飛行」。

### 3.2 成員卡片

每個成員卡片會顯示：

- 名稱（若有）與屬性標籤
- **弱 / 抗 / 免** 各別列出對應倍率（×4, ×2, ×½, ×¼, ×0）
- 若資料庫有種族值：點卡片展開六維種族值長條圖
- **Mega** 按鈕：若該寶可夢有 Mega 進化，此按鈕會出現
  - 每場戰鬥 **限 1 隻** Mega；切換另一隻會自動取消前一個
  - Mega 啟用後屬性、種族值、可學招式、特性會依 Mega 型態更新
- **特性 chip**：列出該寶可夢的所有特性，名稱前有 `✦` 代表「會影響相剋計算」（飄浮、引火、儲水…等）
  - 點擊切換目前啟用的特性；隱藏特性會標 `（隱）`
  - hover 標籤可看到中文說明
  - 已選擇的特性會即時反映在弱抗免區與對戰分析（例：飛行+飄浮 → 地面變 ×0）
- **招式編輯器**：在展開的面板中隨時增減招式屬性（最多 4 個）
- **⋮⋮ 拖曳握把**：長按或按住拖曳即可調整成員順序
- `×` 按鈕：移除該成員

### 3.3 其他操作

- **清空隊伍** — 清除當前隊伍全部成員（不影響其他兩隊）
- **清除選擇** — 清掉新增成員時尚未確認的屬性／招式選擇
- 隊伍頁的「新增成員」區可摺疊，按區塊標題箭頭即可收合

## 4. 隊伍分析

按下 **分析隊伍弱點與攻略** 後，會在下方產生四張分析卡片：

### 4.1 對戰選角（6 選 3）

> 只有當當前隊伍**超過 3 隻**時才出現

- 成員卡片左側會出現一顆 ☐ 勾選框
- 最多可選 3 隻，選好後分析與對手比對就只計算這 3 隻
- 未選滿 3 隻時會退回以全隊計算
- 數字代表選擇順序（1、2、3）

### 4.2 分析卡片

| 卡片 | 內容 |
| --- | --- |
| 🔥 **各成員弱點一覽** | 每隻被剋的屬性與倍率（×2 / ×4），以及「無弱點」標示 |
| ⚔️ **對戰情境分析** | 將威脅屬性分成「危險（≥2 隻被威脅）」與「輕微（只威脅 1 隻）」；危險屬性會列出被威脅的隊員與推薦反擊招式 |
| 🎯 **推薦攜帶的招式屬性** | 依威脅權重排序，挑 6 個涵蓋率最高的屬性 |
| 🚨 **攻擊覆蓋盲點** | 盤點「哪些屬性沒人能打超效」以及「只有 1 人能打超效」；若 18 種都覆蓋完整則顯示 ✅ |

> 攻擊覆蓋計算時：若成員**有設定招式**則以招式為準，否則以成員自身屬性（STAB）估算。

## 5. 即時對手分析

完成隊伍分析後，頁面最下方會出現「🎯 即時對手分析」區塊。

### 輸入對手

**（A）搜尋對手寶可夢**：與屬性查詢相同的搜尋介面。搜到後會顯示對手卡片（屬性、弱抗免、種族值）。

**（B）手動點選屬性**：展開「或手動選擇屬性」可直接點 1～2 個屬性。手動選擇時不會有速度資訊。

### 分析結果

每位隊員一張對比卡片，依「優勢分數」排序（造成傷害 − 受到傷害；同分時速度快者優先）。徽章分類：

| 徽章 | 條件 |
| --- | --- |
| 🟢 **最佳人選** | 能打出 ≥×2 傷害，且自己受傷 ≤×1 |
| 🟢 **能擋住** | 受傷 <×1（抵抗對手）但不一定能打超效 |
| 🟡 **可打但危險** | 能打 ≥×2，但自己也會吃 ≥×2 傷害 |
| 🔴 **避免上場** | 吃 ≥×2 傷害 |
| ⚪ **免疫對手** | 受傷 = 0 |
| ⚪ **普通** | 無特別優勢或劣勢 |

詳情列會標示：

- 對手能打出的最壞倍率
- 自己能打的最佳屬性與倍率
- 若雙方都有種族值：**速度比較**（先手／後手／同速）

底部會附上一排「對付 XX 系的剋制招式屬性」清單（×2 或 ×4）。

### 清除對手

按 **清除對手** 會清掉搜尋欄、手動選擇與比對結果。

## 6. 隊伍匯出與匯入

位於「我的隊伍」分頁頂部：

- **📋 複製隊伍** — 將當前隊伍轉成 JSON 並複製到剪貼簿
- **📥 匯入隊伍** — 彈出視窗，貼上 JSON 後覆蓋當前隊伍

### JSON 格式範例

```json
[
  { "name": "噴火龍", "types": ["fire", "flying"], "moves": ["fire", "flying", "dragon"] },
  { "name": "皮卡丘", "types": ["electric"], "moves": ["electric", "steel"] }
]
```

匯入時會自動驗證屬性 ID、過濾無效資料，並嘗試從內建資料庫補回種族值。

## 7. 資料與儲存

### 屬性倍率

內建 18 種屬性的完整攻防倍率表（`DEFENSE` / `OFFENSE` 矩陣），與官方第 6 代以後的相剋規則一致。

### 寶可夢資料庫

`data/champions-pokemon.json` 收錄：

- 編號、英文名、繁體中文名
- 基本屬性、可學招式屬性（整理後的去重列表）
- 六維種族值（HP / 攻擊 / 防禦 / 特攻 / 特防 / 速度 / 總合）
- 其他型態（Mega、區域形態等）含各自屬性、招式與種族值

### localStorage

- Key：`poke-teams`
- 內容：3 組隊伍的陣列
- 舊版本資料（只有屬性陣列）會在載入時自動升級成新格式 `{ name, types, moves }`

要重置資料：瀏覽器 DevTools → Application → Local Storage → 刪除 `poke-teams` 鍵，或按「清空隊伍」後重新整理。

## 8. 快速開始與安裝

### 方式一：直接開啟（最簡單）

用瀏覽器開啟 `index.html` 即可使用全部功能。

### 方式二：本機伺服器（建議用於行動裝置／PWA）

```bash
# 使用 Node.js 的 serve
npm install -g serve
serve .
# 瀏覽器開啟 http://localhost:3000
```

### 方式三：安裝為 App（PWA）

本專案支援 PWA，可安裝到手機桌面或電腦工作列當原生 App 用，並可離線使用。

> ⚠️ PWA 功能（安裝、Service Worker、離線）**只在 HTTPS 或 localhost 下生效**。直接用 `file://` 開只能用網頁模式。

| 平台 | 安裝方式 |
| --- | --- |
| **iOS Safari** | 點分享 ⬆️ → 加入主畫面（需 iOS 16.4+） |
| **Android Chrome** | 選單 ☰ → 安裝應用程式 |
| **桌面 Chrome / Edge** | 網址列右側出現安裝圖示，點擊即可 |

不安裝也完全沒關係 — 直接當網頁用就好，所有功能完全相同。

### 環境需求

- 任何現代瀏覽器（Chrome、Safari、Edge、Firefox 皆可）
- JavaScript 必須啟用
- 行動裝置與桌面皆支援（RWD 設計）

## 9. 常見問題

**Q. 搜尋找不到某隻寶可夢？**
A. 資料庫目前僅包含《Pokémon Champions》可用陣容。若要手動建立成員，可直接在搜尋欄輸入自訂名稱，然後手動選屬性，不必透過搜尋。

**Q. 我選了招式，為什麼某些屬性是灰色？**
A. 若成員是從資料庫選取的，只允許選該寶可夢能學的招式屬性；若是手動建立的（無名稱或資料庫無此名）則 18 種都可選。

**Q. 「6 選 3」為什麼沒出現？**
A. 只在隊伍成員 > 3 隻時才顯示。少於 4 隻時會直接以全隊分析。

**Q. Mega 進化為何不能多開？**
A. 實際對戰每場限 1 隻 Mega，本工具忠實反映此規則；切換到別的 Mega 會自動取消前一個。

**Q. 隊伍分析出現「攻擊覆蓋完整」是什麼意思？**
A. 表示隊伍中（或對戰選角的 3 隻）的招式屬性合起來，能對全部 18 種屬性打出至少 ×2 傷害。

**Q. 匯入 JSON 失敗？**
A. 確認是合法 JSON 陣列，每筆至少有 `types` 欄位，且屬性 ID 為英文小寫（如 `fire`、`water`）。
